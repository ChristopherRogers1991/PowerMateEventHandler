source code 

Find the PowerMateDevice, and get set up to start reading from it and writing to it.

If the device is not found (can happen if the device is not plugged in, or the user does not have permissions to it) a DeviceNotFound Exception will be raised.

Parameters:
  • brightness (int) - The inital brightness of the led in the base.
  • read_delay (double) - Timeout when waiting for the device to be readable. Having a time out allows the threads to be joinable without waiting for another event. None (default) means to wait indefinitely for the device to be readable. This will probably yield the best performance, but means the thread will not stop after a call to stop() until a new event is triggered.

    Having this configurable was intendted to allow the reading of events to be stoppable (i.e to keep from blocking the thread indefinitely). It was made tunable to allow good performance on fast CPUs, but not hog resources on slower machines.

    Setting delay to None will cause the thread to block indefinitely.

  • turn_delay (double) - Time in ms between consolidated turns.
  • long_press_time (double) - time (in s) the button must be held to register a long press
  • double_click_time (double) - time (in s) the button must be pressed again after a single press to register as a double
  • dev_dir (str) - The directory in which to look for the device.

get_next(self, block=True, timeout=None)

source code 

Pull the next consolidated event off the queue, and return it.

Parameters:
  • block (bool) - block until next is available
  • timeout (double) - block for this long
Returns: evdev.events.InputEvent, ConsolidatedEventCode, or None
If start was run with rawOnly=True, an evdev.events.InputEvent; Otherwise, a ConsolidatedEventCode. In either case, None if there is not an event ready and block is False, or timeout is reached.

Note: block and timeout are passed directly to queue.get(). If block is TRUE, the thread will block for timeout seconds for the next event. If timeout is None, it will wait indefinitely. If block is False, an event will be grabbed only if one is ready immediately.

set_turn_delay(self, delay)

source code 

Set the delay between when consolidated events will be registered.

In an effort to reduce spam from a failry sensative device, this variable was created. If multiple turn events come in, the first will register a consolidated event, and those that come in within the delay time will be ignored. Once the delay threshold has been reached, another consolidated event will be registered.

Parameters:
  • delay (double) - time in ms between turn events.
